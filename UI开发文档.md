# UI架构设计文档

## 项目概述
本项目旨在为抖音视频下载器提供一个图形用户界面（GUI），以替代现有的命令行界面，提升用户体验和易用性。

## 功能需求
- 用户输入抖音用户主页URL或sec_user_id
- 选择视频保存目录
- 提供cURL文件路径（用于cookie认证，可选）
- 实时显示下载进度和状态
- 管理并行下载队列
- 错误处理和详细日志显示
- 支持批量下载和暂停/继续功能

## 架构设计
### 前端层（Presentation Layer）
- **技术栈建议**: 
  - 桌面应用: Python + Tkinter（轻量级、跨平台）
  - Web应用: Python + Flask + HTML/CSS/JavaScript（现代化、可远程访问）
- **主要组件**:
  - 主控制面板: URL输入框、目录选择器、cURL文件选择器、控制按钮
  - 进度监控面板: 视频列表、下载状态、进度条、速度显示
  - 日志显示面板: 实时操作日志、错误信息、调试信息
  - 设置面板: 配置下载参数、并发数、重试策略

### 应用层（Application Layer）
- **服务封装**: 将现有`douyin-downloader`核心逻辑封装为服务接口
- **事件管理**: 处理用户交互事件和下载状态更新
- **队列管理**: 管理下载任务队列和优先级

### 业务逻辑层（Business Logic Layer）
- 复用现有项目组件:
  - `DouyinAPIClient`: API请求和xbogus签名生成
  - `Downloader`: 视频下载和质量选择逻辑
  - `CookieManager`: cookie解析和管理
  - `QueueManager`: 并行下载队列管理

### 数据流设计
1. **用户输入流程**: 前端收集用户输入 → 验证参数 → 调用后端服务
2. **下载流程**: 解析sec_user_id → 获取作品列表 → 队列下载 → 实时状态推送
3. **状态更新**: 后端推送进度 → 前端更新UI → 用户实时监控

## 技术栈建议
### 方案一：桌面应用（推荐）
- **前端**: Tkinter + ttkbootstrap（现代化UI组件）
- **后端**: 现有Python代码直接集成
- **优势**: 部署简单、性能好、无需网络

### 方案二：Web应用
- **前端**: HTML5 + CSS3 + JavaScript（Vue.js/React）
- **后端**: Flask/FastAPI + 现有Python逻辑
- **优势**: 跨平台访问、UI更丰富

## 自动提取功能说明
### 功能概述
新增的自动提取功能可以从cURL文件中自动解析并提取作者主页URL，无需用户手动输入。

### 实现原理
1. **cURL文件解析**: 读取cURL命令文件内容
2. **URL模式匹配**: 使用正则表达式匹配以下模式：
   - `https://www.douyin.com/user/[\w.-]+` (直接的用户主页URL)
   - `https://[^\s]*sec_user_id=[\w-]+` (包含sec_user_id参数的URL)
3. **自动应用**: 当用户未提供URL时，自动使用提取的URL

### 使用场景
1. **简化操作**: 用户只需提供cURL文件，无需手动复制作者主页链接
2. **批量处理**: 支持多个cURL文件自动提取不同作者
3. **错误恢复**: 当URL输入错误时，自动从cURL文件重新提取

### 技术优势
- **准确性**: 直接从浏览器复制的cURL命令确保链接准确性
- **便捷性**: 减少用户操作步骤，提升用户体验
- **兼容性**: 支持各种cURL格式（bash、cmd、PowerShell）
- **容错性**: 自动提取失败时回退到交互式输入

### UI集成建议
- 在cURL文件选择器旁添加"自动提取URL"按钮
- 显示提取状态和结果预览
- 提供手动覆盖自动提取结果的功能
